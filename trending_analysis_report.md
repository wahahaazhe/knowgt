│      ↓                                      │

│      ↓                                      │
│  交易员 → 综合报告，做出决策                 │
│      ↓                                      │
│  风险管理 → 评估风险                        │
│      ↓                                      │
│  投资组合经理 → 最终审批                    │
└─────────────────────────────────────────────┘

## 4. 🔧 JCode - AI 编码代理框架

**仓库**: [1jehuang/jcode](https://github.com/1jehuang/jcode)
**语言**: Rust | **Star**: 3,081 | **今日新增**: +482 ⭐

### 🎯 这个项目是做什么的？

**JCode 是一个高度可定制的 AI 编码代理框架，特点是"有记忆"**。

和普通 AI 编码助手不同：

```
普通 AI 助手:
  你: "帮我写个登录功能"
  AI: (写代码)
  你: "帮我写个注册功能"
  AI: (完全忘记之前写的登录代码，可能风格不一致)

JCode:
  你: "帮我写个登录功能"
  JCode: (写代码 + 存入记忆)
  你: "帮我写个注册功能"
  JCode: (从记忆中取出登录代码风格，保持一致!)
         "我注意到你之前用的是 JWT 认证，我会保持一致"
```

### 💡 核心特点

| 特性 | 说明 |
|------|------|
| **语义记忆** | 自动提取、存储、检索项目上下文 |
| **多代理协作** | 多个 AI 可以同时工作，自动解决冲突 |
| **超低资源** | 仅 27.8MB 内存，14ms 启动 |
| **多 LLM 支持** | Claude、OpenAI、Gemini、Copilot |

### 👥 适合谁用？

- **专业开发者**: 需要长期上下文记忆的项目
- **多任务开发者**: 同时处理多个编码任务
- **追求极致性能**: 看重资源消耗的用户
