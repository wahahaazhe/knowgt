# GitHub Trending 每日运行必看

## 这个项目是干嘛的？

每天自动抓取 GitHub Trending（日/周/月榜），跟历史数据对比，告诉你**今天有什么新变化**——哪些项目新上榜、哪些掉下去了、趋势热点是什么。

## 每日运行流程

### 方式一：自动运行（推荐）

让 Claude Code 每天自动抓取并生成简报。

```bash
# 在 Claude Code 中设置定时任务（Cron）
# 例如：每天上午 9 点自动运行
/CronCreate "0 9 * * *" "抓取 GitHub Trending daily/weekly/monthly，对比历史数据，生成简报写入 trending_reports/"
```

**优点**：零维护、WebFetch 不受反爬限制、AI 直接解读趋势。

### 方式二：手动运行

任何时候想看今天的变化，直接对 Claude Code 说：

> "抓取今天的 GitHub Trending，对比上次记录，告诉我有什么新变化。"

Claude Code 会：
1. 用 `WebFetch` 抓取 `github.com/trending?since=daily/weekly/monthly`
2. 读取 `trending_reports/` 里的历史报告做对比
3. 生成人话版简报（新上榜、掉榜、趋势解读）
4. 把结果写入新的报告文件

## 怎么看历史变化？

Claude Code 会自动对比，但你也可以自己看：

1. **看最新简报**：打开 `trending_reports/` 下日期最新的文件
2. **看连续趋势**：对比最近 2~3 个文件，找重复出现的项目（说明持续火爆）
3. **看掉榜项目**：上次在、这次不在的，说明热度过去了

## 抓取原理

GitHub Trending 页面有反爬限制，直接用 `requests` 容易失败。目前最稳定的方式：

- **Claude Code WebFetch**：直接读取页面内容，不受反爬影响 ✅
- **Playwright/Selenium**：能跑但太重，不推荐日常用
- **requests + BeautifulSoup**：容易超时或被限制 ❌

所以**不要依赖 `github_trending.py` 直接跑**，它只是一个辅助脚本，核心抓取交给 Claude Code 的 WebFetch。

旧 Python 文件已经移动到 `archive/legacy-python/`，保留为历史参考。当前推荐流程是 WebFetch/WebSearch + `trending_reports/` 历史报告 + `github-trending-briefing` skill。
