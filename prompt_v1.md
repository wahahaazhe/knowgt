## 项目讲解提示词（直接用）

分析这个 GitHub Trending 项目，输出要求：

1. 不要出现"架构""底层""范式""赋能""抓手"这类黑话
2. 每个解释必须配一个生活化类比（像给奶奶讲一样）
3. 核心功能只说"它能帮你做什么"，不说技术原理
4. 适合人群要说具体场景，不要写"开发者""工程师"这种笼统词

输出格式：

**这个项目是干嘛的？**（30字内，直接说人话）

**打个比方**（用一个日常物品或场景类比）

**它能帮你做什么？**（列3条，每条开头是动词）
  - 帮你...
  - 让你...
  - 省去...

**谁真的需要它？**（写具体的人，不要写"开发者"）
  - 比如：每天要给老板做PPT的市场专员
  - 比如：接手了别人烂代码的新员工
